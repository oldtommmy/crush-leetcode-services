# Crush LeetCode Services 使用手册

## 1. 服务说明

这是 Crush LeetCode 的服务端仓库。当前主要包含 `apps/mailer`，用于官方周报邮件、内测访问码和后台管理。

为避免公开暴露生产信息，本文档只保留通用使用方式。真实域名、Cloudflare Tunnel、Brevo sender 和密钥只放在部署机器的 `.env.local` 或私有运维记录中。

## 2. 本地启动

```bash
cd apps/mailer
cp .env.example .env.local
npm run start
```

默认监听：

```text
http://127.0.0.1:8787
```

## 3. 环境变量

编辑：

```text
apps/mailer/.env.local
```

关键字段：

```text
BREVO_API_KEY               Brevo API Key
BREVO_FROM_EMAIL            已验证的发件邮箱
BREVO_FROM_NAME             发件人名称
REMINDER_SHARED_SECRET      插件和后端共享密钥
BETA_CODE_SIGNING_SECRET    beta code hash/signature 密钥
BETA_ADMIN_SECRET           本机脚本/API 管理密钥
BETA_ADMIN_USERNAME         后台用户名，默认 admin
BETA_ADMIN_PASSWORD         后台密码
BETA_ADMIN_SESSION_SECRET   后台 session cookie 签名密钥
BETA_DB_PATH                SQLite 路径，默认 .data/mailer.sqlite
PORT                        本机端口，默认 8787
MAILER_BASE_URL             部署后的服务地址
```

不要提交 `.env.local`。

## 4. 后台管理

本地登录页：

```text
http://127.0.0.1:8787/admin
```

默认用户名：

```text
admin
```

密码读取顺序：

```text
BETA_ADMIN_PASSWORD
如果未配置，则兼容使用 BETA_ADMIN_SECRET
```

后台能力：

- 查看用户数、活跃 code、停用用户、累计发送次数。
- 签发官方周报 beta code。
- 查看用户列表和访问码审计表。
- 停用或启用用户。
- 撤销访问码。

注意：

- 明文 code 只在签发成功后显示一次。
- 数据库只保存 code 的 HMAC hash，不保存明文 code。
- 如果用户丢失 code，只能重新签发。

## 5. 健康检查

本机：

```bash
curl -X POST http://127.0.0.1:8787/api/health
```

期望返回：

```json
{
  "ok": true,
  "service": "crush-leetcode-official-mailer",
  "brevoConfigured": true,
  "secretConfigured": true
}
```

生产环境请把域名放在私有部署文档中，不要写进公开 README。

## 6. 签发 Code

推荐使用后台页面。也可以用脚本：

```bash
cd apps/mailer
MAILER_BASE_URL=http://127.0.0.1:8787 bash issue-beta-code.sh user@example.com githubName
```

生产环境使用时把 `MAILER_BASE_URL` 指向真实服务地址。

## 7. 数据库

SQLite 默认路径：

```text
apps/mailer/.data/mailer.sqlite
```

备份示例：

```bash
cp .data/mailer.sqlite .data/mailer.sqlite.backup-$(date +%Y%m%d-%H%M%S)
```

恢复后重启 Node 服务。

## 8. 排障

本机服务不通：

```bash
curl -s -o /dev/null -w '%{http_code}\n' http://127.0.0.1:8787/admin
```

常见原因：

- `npm run start` 没运行。
- `.env.local` 缺少必要密钥。
- 修改 `.env.local` 后没有重启服务。
- Tunnel 或反向代理没有连接到 `127.0.0.1:8787`。

邮件发送失败：

- 检查 `/api/health`。
- 检查 `BREVO_API_KEY` 和 `BREVO_FROM_EMAIL`。
- 检查插件构建时的共享密钥是否和 `REMINDER_SHARED_SECRET` 一致。
- 检查用户填写的收件邮箱是否和 beta code 绑定邮箱一致。

返回 `429`：

- 触发了基础 rate limit。
- 等待响应头 `Retry-After` 指定的秒数后重试。
- 默认限制是单 IP 邮件接口每分钟 30 次、后台登录 5 分钟 10 次、后台管理接口每分钟 120 次。
