# Crush LeetCode 邮件服务使用手册

## 1. 服务说明

这个目录是官方周报邮件服务的本地自托管版本：

```text
/Users/cyh/Desktop/Code/tools/leetcode_plugins/crush_leetcode_mailer_service
```

它负责：

- 接收 Chrome 插件发来的官方周报请求。
- 校验插件共享密钥和用户 beta code。
- 通过 Brevo 发送邮件，发件地址为 `review@crushlc.site`。
- 提供后台管理页，用于签发 code、管理用户、撤销 code。

当前公网地址：

```text
https://mail.crushlc.site
```

## 2. 启动服务

启动 Node 服务：

```bash
cd /Users/cyh/Desktop/Code/tools/leetcode_plugins/crush_leetcode_mailer_service
npm run start
```

本机服务监听：

```text
http://127.0.0.1:8787
```

启动 Cloudflare Tunnel：

```bash
cloudflared tunnel --config /Users/cyh/.cloudflared/crush-leetcode-mailer.yml run
```

Cloudflare Tunnel 会把：

```text
https://mail.crushlc.site -> http://127.0.0.1:8787
```

## 3. 后台登录

登录页：

```text
https://mail.crushlc.site/admin
```

本机登录页：

```text
http://127.0.0.1:8787/admin
```

默认用户名：

```text
admin
```

密码读取顺序：

```text
BETA_ADMIN_PASSWORD
如果未配置，则兼容使用 BETA_ADMIN_SECRET
```

实际值在本机 `.env.local`，不要外发。

登录成功后会写入 HttpOnly session cookie。后台页面不会再把 admin secret 存到 localStorage。

## 4. 后台功能

后台地址：

```text
/admin/dashboard
```

功能：

- 查看用户数、活跃 code、停用用户、累计发送次数。
- 签发官方周报 beta code。
- 查看用户列表和访问码审计表。
- 停用或启用用户。
- 撤销访问码。

注意：

- 明文 code 只在签发成功后显示一次。
- 数据库只保存 code 的 HMAC hash，不保存明文 code。
- 如果用户丢失 code，只能重新签发。

## 5. 常用接口

健康检查：

```bash
curl -X POST https://mail.crushlc.site/api/health
```

期望返回：

```json
{
  "ok": true,
  "service": "crush-leetcode-official-mailer",
  "brevoConfigured": true,
  "secretConfigured": true
}
```

签发 code，推荐直接用后台页面。也可以用脚本：

```bash
cd /Users/cyh/Desktop/Code/tools/leetcode_plugins/crush_leetcode_mailer_service
bash issue-beta-code.sh user@example.com githubName
```

撤销、停用等操作建议在后台页面完成。

## 6. 数据位置

SQLite 数据库：

```text
.data/mailer.sqlite
```

备份示例：

```bash
cp .data/mailer.sqlite .data/mailer.sqlite.backup-$(date +%Y%m%d-%H%M%S)
```

恢复示例：

```bash
cp .data/mailer.sqlite.backup-YYYYMMDD-HHMMSS .data/mailer.sqlite
```

恢复后重启 Node 服务。

## 7. 环境变量

环境变量文件：

```text
.env.local
```

关键字段：

```text
BREVO_API_KEY               Brevo API Key
BREVO_FROM_EMAIL            发件邮箱，当前应为 review@crushlc.site
BREVO_FROM_NAME             发件人名称
REMINDER_SHARED_SECRET      插件和后端共享密钥
BETA_CODE_SIGNING_SECRET    beta code hash/signature 密钥
BETA_ADMIN_SECRET           本机脚本/API 管理密钥
BETA_ADMIN_USERNAME         后台用户名，默认 admin
BETA_ADMIN_PASSWORD         后台密码
BETA_ADMIN_SESSION_SECRET   后台 session cookie 签名密钥
BETA_DB_PATH                SQLite 路径，默认 .data/mailer.sqlite
PORT                        本机端口，默认 8787
MAILER_BASE_URL             公网地址
```

`.env.local` 不要提交到 GitHub。

## 8. 排障

公网出现 Cloudflare `1033`：

```bash
cloudflared tunnel --config /Users/cyh/.cloudflared/crush-leetcode-mailer.yml run
```

先确认本机 Node 服务正常：

```bash
curl -s -o /dev/null -w '%{http_code}\n' http://127.0.0.1:8787/admin
```

后台打不开或登录失败：

- 检查 `npm run start` 是否在运行。
- 检查 `.env.local` 是否有 `BETA_ADMIN_PASSWORD` 或 `BETA_ADMIN_SECRET`。
- 修改 `.env.local` 后必须重启 Node 服务。

邮件发送失败：

- 检查 `/api/health`。
- 检查 `BREVO_API_KEY` 和 `BREVO_FROM_EMAIL`。
- 检查插件构建时的 `VITE_CRUSH_MAILER_SECRET` 是否等于 `REMINDER_SHARED_SECRET`。
- 检查用户填写的收件邮箱是否和 beta code 绑定邮箱一致。

## 9. 插件侧连接

插件默认请求：

```text
https://mail.crushlc.site/api/send-reminder
```

如果后端域名不变，后续后端改实现，插件侧一般不需要改。

构建插件时仍需要注入：

```bash
VITE_CRUSH_MAILER_SECRET=<REMINDER_SHARED_SECRET> npm run build
```
